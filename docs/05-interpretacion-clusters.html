7  Interpretación de clusters

  • Mostrar todo el código
  • Ocultar todo el código
  • Ver el código fuente
Código 
library(tidyverse)
library(knitr)

7.1 Motivación aplicada

Obtener etiquetas de cluster es solo el primer paso. Si no sabemos qué caracteriza a cada grupo, el análisis no es comunicable.

Tip

Nombrar clusters es una decisión interpretativa; debe estar respaldada por variables que realmente diferencien los grupos.

7.2 El clustering no termina con etiquetas

Una etiqueta como cluster 1 no significa nada por sí sola. Hay que describir tamaño, perfil medio, dispersión interna, variables distintivas y relación con etiquetas externas si existen.

Código 
set.seed(123)
vars_mtcars <- mtcars[, c("mpg", "disp", "hp", "wt", "qsec")]
X <- scale(vars_mtcars)
cl <- kmeans(X, centers = 3, nstart = 25)$cluster
kable(table(cl), col.names = c("Cluster", "Frecuencia"))
Cluster Frecuencia
1 6
2 14
3 12

7.3 Perfil medio de cada cluster

Las medias en variables originales son fáciles de comunicar.

Código 
perfiles <- aggregate(vars_mtcars, by = list(cluster = cl), mean)
perfiles |>
  mutate(across(where(is.numeric), function(x) round(x, 2))) |>
  kable()
cluster mpg disp hp wt qsec
1 30.07 86.65 75.50 1.87 18.40
2 15.10 353.10 209.21 4.00 16.77
3 20.92 159.98 109.33 2.98 18.83

Si las variables tenían unidades distintas, las medias estandarizadas ayudan a comparar qué rasgos están por encima o por debajo de la media global.

Código 
Z <- as.data.frame(scale(vars_mtcars))
Z$cluster <- factor(cl)
perfil_z <- aggregate(. ~ cluster, data = Z, mean)
perfil_z_redondeado <- perfil_z
perfil_z_redondeado[, -1] <- round(perfil_z_redondeado[, -1], 2)
kable(perfil_z_redondeado)
cluster mpg disp hp wt qsec
1 1.66 -1.16 -1.04 -1.37 0.31
2 -0.83 0.99 0.91 0.80 -0.60
3 0.14 -0.57 -0.54 -0.25 0.55

7.4 Heatmap de medias estandarizadas

Código 
mat <- as.matrix(perfil_z[, -1])
rownames(mat) <- paste("Cluster", perfil_z$cluster)
as_tibble(mat, rownames = "cluster") |>
  pivot_longer(-cluster, names_to = "variable", values_to = "media_estandarizada") |>
  ggplot(aes(variable, cluster, fill = media_estandarizada)) +
  geom_tile(color = "white") +
  scale_fill_gradient2(low = "#4575b4", mid = "white", high = "#d73027", midpoint = 0) +
  labs(
    title = "Perfil estandarizado por cluster",
    x = "Variable",
    y = "Cluster",
    fill = "Media z"
  ) +
  theme_minimal()

El gráfico resume qué variables diferencian los grupos. Debe leerse junto con tamaños y variabilidad interna.

7.5 Boxplots por cluster

Código 
mtcars_cluster <- vars_mtcars
mtcars_cluster$cluster <- factor(cl)
ggplot(mtcars_cluster, aes(cluster, mpg, fill = cluster)) +
  geom_boxplot(alpha = 0.8, show.legend = FALSE) +
  labs(title = "Distribución por cluster", x = "Cluster", y = "Millas por galón") +
  theme_minimal()

Los boxplots muestran si la diferencia entre perfiles medios es estable o si hay solapamiento.

7.6 Comparación con etiquetas externas

Cuando existe una etiqueta externa, se puede cruzar con los clusters.

Código 
kable(table(cluster = cl, transmision = factor(mtcars$am, labels = c("automatico", "manual"))))
automatico manual
0 6
12 2
7 5
Código 
kable(table(cluster = cl, cilindros = mtcars$cyl))
4 6 8
6 0 0
0 0 14
5 7 0

Esta tabla no valida automáticamente el clustering, pero ayuda a interpretar qué estructura conocida captura.

7.7 Nombrar clusters

Un nombre útil debe ser breve y estar justificado. En mtcars, por ejemplo, podríamos hablar de coches ligeros y eficientes frente a coches pesados y potentes si las medias lo sostienen.

Código 
aggregate(vars_mtcars, by = list(cluster = cl), mean) |>
  mutate(across(where(is.numeric), function(x) round(x, 2))) |>
  kable()
cluster mpg disp hp wt qsec
1 30.07 86.65 75.50 1.87 18.40
2 15.10 353.10 209.21 4.00 16.77
3 20.92 159.98 109.33 2.98 18.83

7.8 Estabilidad e incertidumbre

Los clusters pueden cambiar con pequeñas modificaciones: otra semilla, otra selección de variables o una observación atípica. Una comprobación sencilla es repetir k-means con varias semillas y comparar tamaños o tablas cruzadas.

Código 
set.seed(1)
res <- replicate(10, kmeans(X, centers = 3, nstart = 1)$tot.withinss)
tibble(ejecucion = seq_along(res), tot_withinss = round(res, 1)) |>
  kable()
ejecucion tot_withinss
1 56.4
2 53.2
3 54.1
4 53.2
5 52.9
6 54.1
7 53.5
8 54.1
9 52.9
10 56.7

7.9 Mini-informe de clustering

Un informe mínimo debería incluir:

  • pregunta aplicada;
  • variables activas y motivo de escalado;
  • distancia o algoritmo;
  • criterio para elegir número de clusters;
  • perfil de cada cluster;
  • limitaciones y advertencias.

7.10 Errores frecuentes

  • Describir clusters solo con el algoritmo usado.
  • Nombrar grupos a partir de una única variable.
  • Ignorar clusters pequeños.
  • Presentar etiquetas de cluster como clases naturales sin evidencia adicional.

7.11 Resumen del capítulo

Interpretar clusters exige volver a los datos originales, caracterizar perfiles y reconocer incertidumbre. La etiqueta de cluster es una ayuda, no una conclusión completa.

7.12 Ejercicios propuestos

  1. Crea un perfil de clusters para mtcars usando otro subconjunto de variables.
  2. Construye una tabla cluster por transmisión (am) y comenta si ayuda a interpretar los grupos.
  3. Redacta tres nombres posibles para clusters de mtcars y justifícalos con medias.
