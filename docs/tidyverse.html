15.1 Objetivo del capítulo

En los apuntes usamos mucho tidyverse porque permite escribir análisis de datos como una secuencia legible de transformaciones. Este capítulo resume las funciones más frecuentes, especialmente las que aparecen en los ejemplos del libro.

La idea central es trabajar con tablas y encadenar pasos:

  1. elegir variables;
  2. filtrar filas;
  3. crear o transformar columnas;
  4. agrupar;
  5. resumir;
  6. ordenar;
  7. representar o mostrar resultados.

Usaremos mtcars como tabla principal.

Código 
coches <- mtcars |>
  as_tibble(rownames = "modelo") |>
  mutate(
    transmision = factor(am, labels = c("automatico", "manual")),
    cilindros = factor(cyl)
  )

coches |>
  select(modelo, mpg, disp, hp, wt, transmision, cilindros) |>
  head(6) |>
  kable()
modelo mpg disp hp wt transmision cilindros
Mazda RX4 21.0 160 110 2.620 manual 6
Mazda RX4 Wag 21.0 160 110 2.875 manual 6
Datsun 710 22.8 108 93 2.320 manual 4
Hornet 4 Drive 21.4 258 110 3.215 automatico 6
Hornet Sportabout 18.7 360 175 3.440 automatico 8
Valiant 18.1 225 105 3.460 automatico 6

15.2 La tubería

La tubería |> lee el código de izquierda a derecha. El resultado de una línea entra como primer argumento de la función siguiente.

Código 
coches |>
  select(modelo, mpg, wt, transmision) |>
  filter(transmision == "manual") |>
  arrange(desc(mpg)) |>
  head(5) |>
  kable()
modelo mpg wt transmision
Toyota Corolla 33.9 1.835 manual
Fiat 128 32.4 2.200 manual
Honda Civic 30.4 1.615 manual
Lotus Europa 30.4 1.513 manual
Fiat X1-9 27.3 1.935 manual

Sin tuberías, el mismo código sería más anidado y menos legible:

Código 
resultado <- head(
  arrange(
    filter(
      select(coches, modelo, mpg, wt, transmision),
      transmision == "manual"
    ),
    desc(mpg)
  ),
  5
)

kable(resultado)
modelo mpg wt transmision
Toyota Corolla 33.9 1.835 manual
Fiat 128 32.4 2.200 manual
Honda Civic 30.4 1.615 manual
Lotus Europa 30.4 1.513 manual
Fiat X1-9 27.3 1.935 manual

En estos apuntes usamos principalmente |>, la tubería nativa de R. También existe %>%, del paquete magrittr, muy común en código tidyverse más antiguo. Para la mayoría de usos básicos, ambas permiten escribir cadenas de transformación similares.

15.3 select(): elegir columnas

select() conserva, reordena o descarta variables.

Código 
coches |>
  select(modelo, mpg, hp, wt) |>
  head(5) |>
  kable()
modelo mpg hp wt
Mazda RX4 21.0 110 2.620
Mazda RX4 Wag 21.0 110 2.875
Datsun 710 22.8 93 2.320
Hornet 4 Drive 21.4 110 3.215
Hornet Sportabout 18.7 175 3.440

También se pueden usar ayudantes de selección:

Código 
coches |>
  select(modelo, starts_with("d"), ends_with("s")) |>
  head(5) |>
  kable()
modelo disp drat vs cilindros
Mazda RX4 160 3.90 0 6
Mazda RX4 Wag 160 3.90 0 6
Datsun 710 108 3.85 1 4
Hornet 4 Drive 258 3.08 1 6
Hornet Sportabout 360 3.15 0 8

Para quitar columnas se usa el signo menos.

Código 
coches |>
  select(-modelo, -transmision, -cilindros) |>
  head(5) |>
  kable()
mpg cyl disp hp drat wt qsec vs am gear carb
21.0 6 160 110 3.90 2.620 16.46 0 1 4 4
21.0 6 160 110 3.90 2.875 17.02 0 1 4 4

18.7 8 360 175 3.15 3.440 17.02 0 0 3 2

En análisis multivariante, select() es útil para separar variables activas y suplementarias. Por ejemplo, podemos construir una matriz numérica activa y guardar la transmisión para interpretar después.

X <- coches |>
  select(mpg, disp, hp, wt, qsec)

etiqueta <- coches |>
  pull(transmision)

kable(head(X, 5))

15.4 filter(): filtrar filas

filter() conserva observaciones que cumplen una condición lógica.

Código 
coches |>
  filter(mpg > 25) |>
  select(modelo, mpg, hp, wt, transmision) |>
  kable()
modelo mpg hp wt transmision
Fiat 128 32.4 66 2.200 manual
Honda Civic 30.4 52 1.615 manual
Toyota Corolla 33.9 65 1.835 manual
Fiat X1-9 27.3 66 1.935 manual
Porsche 914-2 26.0 91 2.140 manual
Lotus Europa 30.4 113 1.513 manual

Se pueden combinar condiciones con & y |.

Código 
coches |>
  filter(mpg > 20, wt < 3) |>
  select(modelo, mpg, wt, cilindros, transmision) |>
  kable()
modelo mpg wt cilindros transmision
Mazda RX4 21.0 2.620 6 manual
Mazda RX4 Wag 21.0 2.875 6 manual
Datsun 710 22.8 2.320 4 manual
Fiat 128 32.4 2.200 4 manual
Honda Civic 30.4 1.615 4 manual
Toyota Corolla 33.9 1.835 4 manual
Toyota Corona 21.5 2.465 4 automatico
Fiat X1-9 27.3 1.935 4 manual
Porsche 914-2 26.0 2.140 4 manual
Lotus Europa 30.4 1.513 4 manual
Volvo 142E 21.4 2.780 4 manual

La coma dentro de filter() equivale a una condición conjunta: deben cumplirse todas las condiciones. Para alternativas usamos |.

Código 
coches |>
  filter(cilindros == "4" | transmision == "manual") |>
  count(cilindros, transmision) |>
  kable()
cilindros transmision n
4 automatico 3
4 manual 8
6 manual 3
8 manual 2

15.6 mutate(): crear columnas

mutate() añade nuevas variables o modifica variables existentes.

Código 
coches |>
  mutate(
    peso_kg_aprox = wt * 1000 * 0.453592,
    consumo_relativo = mpg / mean(mpg)
  ) |>
  select(modelo, wt, peso_kg_aprox, mpg, consumo_relativo) |>
  head(5) |>
  kable(digits = 2)
modelo wt peso_kg_aprox mpg consumo_relativo
Mazda RX4 2.62 1188.41 21.0 1.05
Mazda RX4 Wag 2.88 1304.08 21.0 1.05
Datsun 710 2.32 1052.33 22.8 1.13
Hornet 4 Drive 3.21 1458.30 21.4 1.07
Hornet Sportabout 3.44 1560.36 18.7 0.93

Es frecuente usar mutate() para crear factores interpretables a partir de códigos numéricos.

Código 
coches |>
  mutate(
    tipo_motor = factor(vs, labels = c("V", "Straight")),
    potencia = if_else(hp >= median(hp), "alta", "baja")
  ) |>
  count(tipo_motor, potencia) |>
  kable()

15.7 case_when(): crear categorías con varias reglas

case_when() es útil cuando una nueva variable depende de varias condiciones.

Código 
coches |>
  mutate(
    perfil_consumo = case_when(
      mpg >= 25 ~ "eficiente",
      mpg >= 18 ~ "intermedio",
      TRUE ~ "alto consumo"
    )
  ) |>
  count(perfil_consumo) |>
  kable()
perfil_consumo n
alto consumo 13
eficiente 6
intermedio 13

El caso TRUE ~ ... funciona como opción por defecto.

15.8 summarise() y group_by(): resumir

summarise() reduce una tabla a estadísticas resumen.

Código 
coches |>
  summarise(
    n = n(),
    media_mpg = mean(mpg),
    sd_mpg = sd(mpg),
    media_hp = mean(hp)
  ) |>
  kable(digits = 2)
n media_mpg sd_mpg media_hp
32 20.09 6.03 146.69

Con group_by() el resumen se calcula por grupos.

Código 
coches |>
  group_by(transmision) |>
  summarise(
    n = n(),
    media_mpg = mean(mpg),
    media_hp = mean(hp),
    media_wt = mean(wt),
    .groups = "drop"
  ) |>
  kable(digits = 2)
transmision n media_mpg media_hp media_wt
automatico 19 17.15 160.26 3.77
manual 13 24.39 126.85 2.41

Este patrón aparece mucho al interpretar clusters: primero asignamos una etiqueta de grupo y después calculamos medias por grupo.

Código 
set.seed(123)
clusters <- coches |>
  select(mpg, disp, hp, wt) |>
  scale() |>
  kmeans(centers = 3, nstart = 25)

coches |>
  mutate(cluster = factor(clusters$cluster)) |>
  group_by(cluster) |>
  summarise(
    n = n(),
    mpg = mean(mpg),
    disp = mean(disp),
    hp = mean(hp),
    wt = mean(wt),
    .groups = "drop"
  ) |>
  kable(digits = 2)
cluster n mpg disp hp wt
1 14 15.10 353.10 209.21 4.00
2 6 30.07 86.65 75.50 1.87
3 12 20.92 159.98 109.33 2.98

15.9 count(): contar combinaciones

count() es una abreviatura de group_by() seguido de summarise(n = n()).

Código 
coches |>
  count(cilindros, transmision) |>
  kable()
cilindros transmision n
4 automatico 3
4 manual 8
6 automatico 4
6 manual 3
8 automatico 12
8 manual 2

Puede ordenarse por frecuencia:

Código 
coches |>
  count(cilindros, transmision, sort = TRUE) |>
  kable()
cilindros transmision n
8 automatico 12
4 manual 8
6 automatico 4
4 automatico 3
6 manual 3
8 manual 2

En capítulos de correspondencias y MCA usamos con frecuencia tablas de conteos porque la información categórica se resume mediante frecuencias.

15.10 across(): aplicar lo mismo a varias columnas

across() permite aplicar una función a varias variables dentro de summarise() o mutate().

Código 
coches |>
  summarise(
    across(c(mpg, disp, hp, wt), mean)
  ) |>
  kable(digits = 2)
mpg disp hp wt
20.09 230.72 146.69 3.22

También podemos combinar group_by() y across().

Código 
coches |>
  group_by(transmision) |>
  summarise(
    across(c(mpg, disp, hp, wt), mean),
    .groups = "drop"
  ) |>
  kable(digits = 2)
transmision mpg disp hp wt
automatico 17.15 290.38 160.26 3.77
manual 24.39 143.53 126.85 2.41

Para escalar varias variables dentro de una tabla, puede usarse:

Código 
coches_escalados <- coches |>
  mutate(
    across(c(mpg, disp, hp, wt), function(x) as.numeric(scale(x)))
  )

coches_escalados |>
  select(modelo, mpg, disp, hp, wt) |>
  head(5) |>
  kable(digits = 2)
modelo mpg disp hp wt
Mazda RX4 0.15 -0.57 -0.54 -0.61
Mazda RX4 Wag 0.15 -0.57 -0.54 -0.35
Datsun 710 0.45 -0.99 -0.78 -0.92
Hornet 4 Drive 0.22 0.22 -0.54 0.00
Hornet Sportabout -0.23 1.04 0.41 0.23

Para análisis multivariante suele ser más claro construir directamente la matriz escalada con scale(), pero across() es cómodo cuando queremos conservar la tabla completa.

15.11 pull(): extraer una columna como vector

pull() extrae una columna de una tabla. Es útil cuando una función espera un vector.

Código 
transmision <- coches |>
  pull(transmision)

table(transmision) |>
  kable(col.names = c("Transmisión", "Frecuencia"))
Transmisión Frecuencia
automatico 19
manual 13

En los apuntes aparece esta idea al guardar una variable suplementaria, por ejemplo una etiqueta externa para interpretar clusters o ejes factoriales.

15.13 pivot_longer(): formato largo

Muchas gráficas con ggplot2 funcionan mejor en formato largo: una columna indica la variable y otra el valor.

Código 
coches_largo <- coches |>
  select(modelo, transmision, mpg, hp, wt) |>
  pivot_longer(
    cols = c(mpg, hp, wt),
    names_to = "variable",
    values_to = "valor"
  )

coches_largo |>
  head(8) |>
  kable()
modelo transmision variable valor
Mazda RX4 manual mpg 21.000
Mazda RX4 manual hp 110.000
Mazda RX4 manual wt 2.620
Mazda RX4 Wag manual mpg 21.000
Mazda RX4 Wag manual hp 110.000
Mazda RX4 Wag manual wt 2.875
Datsun 710 manual mpg 22.800
Datsun 710 manual hp 93.000

Esto permite hacer una misma gráfica para varias variables.

Código 
coches_largo |>
  ggplot(aes(transmision, valor, fill = transmision)) +
  geom_boxplot(alpha = 0.75, show.legend = FALSE) +
  facet_wrap(~ variable, scales = "free_y") +
  labs(x = "Transmisión", y = "Valor") +
  theme_minimal()

15.14 pivot_wider(): formato ancho

pivot_wider() hace la operación inversa: reparte valores de una columna en varias columnas.

Código 
resumen_largo <- coches |>
  group_by(transmision) |>
  summarise(
    mpg = mean(mpg),
    hp = mean(hp),
    wt = mean(wt),
    .groups = "drop"
  ) |>
  pivot_longer(
    cols = c(mpg, hp, wt),
    names_to = "variable",
    values_to = "media"
  )

resumen_largo |>
  pivot_wider(
    names_from = transmision,
    values_from = media
  ) |>
  kable(digits = 2)
variable automatico manual
mpg 17.15 24.39
hp 160.26 126.85
wt 3.77 2.41

15.15 left_join(): unir tablas

Los join combinan tablas usando una o varias claves. En análisis aplicado esto aparece al unir resultados de un modelo con información original.

Código 
coordenadas_pca <- coches |>
  select(mpg, disp, hp, wt) |>
  prcomp(center = TRUE, scale. = TRUE) |>
  pluck("x") |>
  as_tibble() |>
  select(PC1, PC2) |>
  mutate(modelo = coches$modelo)

etiquetas <- coches |>
  select(modelo, transmision, cilindros)

coordenadas_pca |>
  left_join(etiquetas, by = "modelo") |>
  head(6) |>
  kable(digits = 2)
PC1 PC2 modelo transmision cilindros
-0.93 0.03 Mazda RX4 manual 6
-0.80 0.17 Mazda RX4 Wag manual 6
-1.57 -0.02 Datsun 710 manual 4
-0.25 0.44 Hornet 4 Drive automatico 6
0.96 -0.08 Hornet Sportabout automatico 8
-0.02 0.67 Valiant automatico 6

left_join(x, y, by = "clave") conserva todas las filas de x y añade las columnas de y cuando encuentra coincidencias.

15.16 ggplot() dentro de pipelines

ggplot2 también encaja bien con tuberías. Primero transformamos la tabla y luego la enviamos a ggplot().

Código 
coches |>
  mutate(peso = if_else(wt >= median(wt), "pesado", "ligero")) |>
  ggplot(aes(wt, mpg, color = peso, shape = transmision)) +
  geom_point(size = 2.4, alpha = 0.85) +
  labs(
    title = "Consumo y peso en mtcars",
    x = "Peso",
    y = "Millas por galón",
    color = "Peso relativo",
    shape = "Transmisión"
  ) +
  theme_minimal()

Una regla práctica: si una gráfica necesita una tabla transformada, conviene preparar esa tabla con select(), filter(), mutate() o summarise() antes de llamar a ggplot().

15.18 Errores frecuentes

  • Usar select() cuando se quieren filtrar filas; para filas se usa filter().
  • Olvidar que filter() necesita condiciones lógicas, como mpg > 20.
  • Crear una variable con mutate() pero no guardar el resultado ni continuar la tubería.
  • Resumir después de group_by() y olvidar .groups = "drop" cuando se quiere una tabla sin agrupación.
  • Tratar códigos categóricos como números sin convertirlos a factor.
  • Perder una variable suplementaria antes de usarla para interpretar resultados.

15.19 Resumen del capítulo

El flujo tidyverse se basa en transformar tablas de forma explícita y legible. Para los apuntes, las funciones más importantes son select(), filter(), mutate(), arrange(), group_by(), summarise(), count(), across(), pivot_longer(), pivot_wider(), pull() y los join. ## Objetivo del capítulo

En los apuntes usamos mucho `tidyverse` porque permite escribir análisis de datos como una secuencia legible de transformaciones. Este capítulo resume las funciones más frecuentes, especialmente las que aparecen en los ejemplos del libro.

La idea central es trabajar con tablas y encadenar pasos:

1. elegir variables;
2. filtrar filas;
3. crear o transformar columnas;
4. agrupar;
5. resumir;
6. ordenar;
7. representar o mostrar resultados.

Usaremos `mtcars` como tabla principal.

```{r}
coches <- mtcars |>
  as_tibble(rownames = "modelo") |>
  mutate(
    transmision = factor(am, labels = c("automatico", "manual")),
    cilindros = factor(cyl)
  )

coches |>
  select(modelo, mpg, disp, hp, wt, transmision, cilindros) |>
  head(6) |>
  kable()
```

## La tubería

La tubería `|>` lee el código de izquierda a derecha. El resultado de una línea entra como primer argumento de la función siguiente.

```{r}
coches |>
  select(modelo, mpg, wt, transmision) |>
  filter(transmision == "manual") |>
  arrange(desc(mpg)) |>
  head(5) |>
  kable()
```

Sin tuberías, el mismo código sería más anidado y menos legible:

```{r}
resultado <- head(
  arrange(
    filter(
      select(coches, modelo, mpg, wt, transmision),
      transmision == "manual"
    ),
    desc(mpg)
  ),
  5
)

kable(resultado)
```

En estos apuntes usamos principalmente `|>`, la tubería nativa de R. También existe `%>%`, del paquete `magrittr`, muy común en código tidyverse más antiguo. Para la mayoría de usos básicos, ambas permiten escribir cadenas de transformación similares.

## `select()`: elegir columnas

`select()` conserva, reordena o descarta variables.

```{r}
coches |>
  select(modelo, mpg, hp, wt) |>
  head(5) |>
  kable()
```

También se pueden usar ayudantes de selección:

```{r}
coches |>
  select(modelo, starts_with("d"), ends_with("s")) |>
  head(5) |>
  kable()
```

Para quitar columnas se usa el signo menos.

```{r}
coches |>
  select(-modelo, -transmision, -cilindros) |>
  head(5) |>
  kable()
```

En análisis multivariante, `select()` es útil para separar variables activas y suplementarias. Por ejemplo, podemos construir una matriz numérica activa y guardar la transmisión para interpretar después.

```{r}
X <- coches |>
  select(mpg, disp, hp, wt, qsec)

etiqueta <- coches |>
  pull(transmision)

kable(head(X, 5))
```

## `filter()`: filtrar filas

`filter()` conserva observaciones que cumplen una condición lógica.

```{r}
coches |>
  filter(mpg > 25) |>
  select(modelo, mpg, hp, wt, transmision) |>
  kable()
```

Se pueden combinar condiciones con `&` y `|`.

```{r}
coches |>
  filter(mpg > 20, wt < 3) |>
  select(modelo, mpg, wt, cilindros, transmision) |>
  kable()
```

La coma dentro de `filter()` equivale a una condición conjunta: deben cumplirse todas las condiciones. Para alternativas usamos `|`.

```{r}
coches |>
  filter(cilindros == "4" | transmision == "manual") |>
  count(cilindros, transmision) |>
  kable()
```

## `arrange()`: ordenar filas

`arrange()` ordena una tabla según una o varias columnas.

```{r}
coches |>
  arrange(mpg) |>
  select(modelo, mpg, wt, hp) |>
  head(5) |>
  kable()
```

Para ordenar de mayor a menor se usa `desc()`.

```{r}
coches |>
  arrange(desc(hp)) |>
  select(modelo, hp, mpg, wt) |>
  head(5) |>
  kable()
```

## `mutate()`: crear columnas

`mutate()` añade nuevas variables o modifica variables existentes.

```{r}
coches |>
  mutate(
    peso_kg_aprox = wt * 1000 * 0.453592,
    consumo_relativo = mpg / mean(mpg)
  ) |>
  select(modelo, wt, peso_kg_aprox, mpg, consumo_relativo) |>
  head(5) |>
  kable(digits = 2)
```

Es frecuente usar `mutate()` para crear factores interpretables a partir de códigos numéricos.

```{r}
coches |>
  mutate(
    tipo_motor = factor(vs, labels = c("V", "Straight")),
    potencia = if_else(hp >= median(hp), "alta", "baja")
  ) |>
  count(tipo_motor, potencia) |>
  kable()
```

## `case_when()`: crear categorías con varias reglas

`case_when()` es útil cuando una nueva variable depende de varias condiciones.

```{r}
coches |>
  mutate(
    perfil_consumo = case_when(
      mpg >= 25 ~ "eficiente",
      mpg >= 18 ~ "intermedio",
      TRUE ~ "alto consumo"
    )
  ) |>
  count(perfil_consumo) |>
  kable()
```

El caso `TRUE ~ ...` funciona como opción por defecto.

## `summarise()` y `group_by()`: resumir

`summarise()` reduce una tabla a estadísticas resumen.

```{r}
coches |>
  summarise(
    n = n(),
    media_mpg = mean(mpg),
    sd_mpg = sd(mpg),
    media_hp = mean(hp)
  ) |>
  kable(digits = 2)
```

Con `group_by()` el resumen se calcula por grupos.

```{r}
coches |>
  group_by(transmision) |>
  summarise(
    n = n(),
    media_mpg = mean(mpg),
    media_hp = mean(hp),
    media_wt = mean(wt),
    .groups = "drop"
  ) |>
  kable(digits = 2)
```

Este patrón aparece mucho al interpretar clusters: primero asignamos una etiqueta de grupo y después calculamos medias por grupo.

```{r}
set.seed(123)
clusters <- coches |>
  select(mpg, disp, hp, wt) |>
  scale() |>
  kmeans(centers = 3, nstart = 25)

coches |>
  mutate(cluster = factor(clusters$cluster)) |>
  group_by(cluster) |>
  summarise(
    n = n(),
    mpg = mean(mpg),
    disp = mean(disp),
    hp = mean(hp),
    wt = mean(wt),
    .groups = "drop"
  ) |>
  kable(digits = 2)
```

## `count()`: contar combinaciones

`count()` es una abreviatura de `group_by()` seguido de `summarise(n = n())`.

```{r}
coches |>
  count(cilindros, transmision) |>
  kable()
```

Puede ordenarse por frecuencia:

```{r}
coches |>
  count(cilindros, transmision, sort = TRUE) |>
  kable()
```

En capítulos de correspondencias y MCA usamos con frecuencia tablas de conteos porque la información categórica se resume mediante frecuencias.

## `across()`: aplicar lo mismo a varias columnas

`across()` permite aplicar una función a varias variables dentro de `summarise()` o `mutate()`.

```{r}
coches |>
  summarise(
    across(c(mpg, disp, hp, wt), mean)
  ) |>
  kable(digits = 2)
```

También podemos combinar `group_by()` y `across()`.

```{r}
coches |>
  group_by(transmision) |>
  summarise(
    across(c(mpg, disp, hp, wt), mean),
    .groups = "drop"
  ) |>
  kable(digits = 2)
```

Para escalar varias variables dentro de una tabla, puede usarse:

```{r}
coches_escalados <- coches |>
  mutate(
    across(c(mpg, disp, hp, wt), function(x) as.numeric(scale(x)))
  )

coches_escalados |>
  select(modelo, mpg, disp, hp, wt) |>
  head(5) |>
  kable(digits = 2)
```

Para análisis multivariante suele ser más claro construir directamente la matriz escalada con `scale()`, pero `across()` es cómodo cuando queremos conservar la tabla completa.

## `pull()`: extraer una columna como vector

`pull()` extrae una columna de una tabla. Es útil cuando una función espera un vector.

```{r}
transmision <- coches |>
  pull(transmision)

table(transmision) |>
  kable(col.names = c("Transmisión", "Frecuencia"))
```

En los apuntes aparece esta idea al guardar una variable suplementaria, por ejemplo una etiqueta externa para interpretar clusters o ejes factoriales.

## `slice()`: elegir filas por posición

`slice()` selecciona filas por posición. Algunas variantes útiles son `slice_head()`, `slice_max()` y `slice_sample()`.

```{r}
coches |>
  slice_head(n = 4) |>
  select(modelo, mpg, hp, wt) |>
  kable()
```

```{r}
coches |>
  slice_max(hp, n = 5) |>
  select(modelo, hp, mpg, wt) |>
  kable()
```

```{r}
set.seed(42)
coches |>
  slice_sample(n = 5) |>
  select(modelo, mpg, hp, wt, transmision) |>
  kable()
```

## `pivot_longer()`: formato largo

Muchas gráficas con `ggplot2` funcionan mejor en formato largo: una columna indica la variable y otra el valor.

```{r}
coches_largo <- coches |>
  select(modelo, transmision, mpg, hp, wt) |>
  pivot_longer(
    cols = c(mpg, hp, wt),
    names_to = "variable",
    values_to = "valor"
  )

coches_largo |>
  head(8) |>
  kable()
```

Esto permite hacer una misma gráfica para varias variables.

```{r}
coches_largo |>
  ggplot(aes(transmision, valor, fill = transmision)) +
  geom_boxplot(alpha = 0.75, show.legend = FALSE) +
  facet_wrap(~ variable, scales = "free_y") +
  labs(x = "Transmisión", y = "Valor") +
  theme_minimal()
```

## `pivot_wider()`: formato ancho

`pivot_wider()` hace la operación inversa: reparte valores de una columna en varias columnas.

```{r}
resumen_largo <- coches |>
  group_by(transmision) |>
  summarise(
    mpg = mean(mpg),
    hp = mean(hp),
    wt = mean(wt),
    .groups = "drop"
  ) |>
  pivot_longer(
    cols = c(mpg, hp, wt),
    names_to = "variable",
    values_to = "media"
  )

resumen_largo |>
  pivot_wider(
    names_from = transmision,
    values_from = media
  ) |>
  kable(digits = 2)
```

## `left_join()`: unir tablas

Los `join` combinan tablas usando una o varias claves. En análisis aplicado esto aparece al unir resultados de un modelo con información original.

```{r}
coordenadas_pca <- coches |>
  select(mpg, disp, hp, wt) |>
  prcomp(center = TRUE, scale. = TRUE) |>
  pluck("x") |>
  as_tibble() |>
  select(PC1, PC2) |>
  mutate(modelo = coches$modelo)

etiquetas <- coches |>
  select(modelo, transmision, cilindros)

coordenadas_pca |>
  left_join(etiquetas, by = "modelo") |>
  head(6) |>
  kable(digits = 2)
```

`left_join(x, y, by = "clave")` conserva todas las filas de `x` y añade las columnas de `y` cuando encuentra coincidencias.

## `ggplot()` dentro de pipelines

`ggplot2` también encaja bien con tuberías. Primero transformamos la tabla y luego la enviamos a `ggplot()`.

```{r}
coches |>
  mutate(peso = if_else(wt >= median(wt), "pesado", "ligero")) |>
  ggplot(aes(wt, mpg, color = peso, shape = transmision)) +
  geom_point(size = 2.4, alpha = 0.85) +
  labs(
    title = "Consumo y peso en mtcars",
    x = "Peso",
    y = "Millas por galón",
    color = "Peso relativo",
    shape = "Transmisión"
  ) +
  theme_minimal()
```

Una regla práctica: si una gráfica necesita una tabla transformada, conviene preparar esa tabla con `select()`, `filter()`, `mutate()` o `summarise()` antes de llamar a `ggplot()`.

## Errores frecuentes

- Usar `select()` cuando se quieren filtrar filas; para filas se usa `filter()`.
- Olvidar que `filter()` necesita condiciones lógicas, como `mpg > 20`.
- Crear una variable con `mutate()` pero no guardar el resultado ni continuar la tubería.
- Resumir después de `group_by()` y olvidar `.groups = "drop"` cuando se quiere una tabla sin agrupación.
- Tratar códigos categóricos como números sin convertirlos a `factor`.
- Perder una variable suplementaria antes de usarla para interpretar resultados.
